What you'll learn

The objective of the following labs is to understand the implementation of Continuous Integration (CI) in a real scenario example in regards to the continuous merging of changes into a shared version control repository.

We'll answer common questions like:

  • What is Continuous Integration?
  • How can Continuous Integration be implemented in a real scenario using GitHub?
  • What is codespace?
  • What is a workflow file and how are workflows used with GitHub Actions?
  • How to setup static code analysis using GitHub?
  • How to setup branch policies in GitHub?

And when you're done you'll be able to:

  1. Push code changes to a local branch for revision and integration into the main branch.
  2. Configuring the build pipeline to build the application and run unit tests.
  3. Making use of code analysis features available out of the box in GitHub.
  4. Configuring the branch policies for main branch.

What you'll build

In this lab you will use codespaces to build and test your application. Next, you will learn how to setup a build pipeline for continuous integration. You will also learn about the static code analysis tools available in GitHub. Next you will learn how to configure branch policies in GitHub.

Prerequisites

Before you start, you should be familiar with GitHub and Continuous Integration and must have completed the previous labs. We recommend taking a look to the following resources and Learning Labs prior to this course:

  • Introduction to GitHub
  • Hello GitHub Actions!
  • Continuous Integration with GitHub Actions
  • Analyze DevOps Continuous Planning and Continuous Integration (only Continuous Integration modules)
