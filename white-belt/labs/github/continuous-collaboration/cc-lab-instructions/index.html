Overview

This lab is based on SmartHotel360, which is a fictitious smart hospitality company. Due to the complexity of managing the local infrastructure, SmartHotel360 wants to re-host their web app with minimal efforts in Azure. In the Continuous Collaboration lab we demonstrate how Teams as a collaboration tool can be used in sync with GitHub and Azure Boards to receive updates for various operations across GitHub and Azure Boards.

Context

Collaboration is very important in any teamwork. Microsoft Teams is a great tool to achieve this, and with various apps available out of the box, collaboration can be more efficient and optimized.

  • Understanding of how GitHub and Azure Boards App can help increase productivity by receiving updates right into teams channel.
  • Understanding of how Azure Boards and GitHub can be connected to link the work items with project code.

What?

Continuous collaboration is a practice that brings teams and members to work together to solve problems that require close interactions and quick actions. In this lab, we will learn how an effective collaboration environment can be created by using GitHub, Azure Boards and Microsoft Teams in sync with one another.

Why?

Continuous Collaboration emphasizes the communication aspects in a team that will either make or break the success of the team. Gaining insight and skills in these aspects brings an advantage, making it easier to work and trust team members.

How?

To enable Continuous Collaboration, we will make use of two Microsoft Teams applications: GitHub App & Azure Boards App. Along with this we use the out of box features provided by Azure Boards to link the project code in GitHub with work items in Azure Boards.
